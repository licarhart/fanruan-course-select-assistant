# 选课助手 · AI 选课决策助手

> 面向高校学生的 **AI 选课决策助手**：前台是评课社区底座，其上叠加一层"选课决策助手"。
> 用 AI 把散落、口语化的课程评价**结构化**成可比较、可个性化的数据，结合客观选课/给分数据，
> 为不同目标的学生（保研 / 留学 / 考研 / 求通过…）给出**可解释**的选课建议。

这是一个 AI 提效产品方案 + 可运行 demo。本仓库收录对外可公开的精华成果：调研、产品设计、运营方案、AI 实验，以及一个纯前端可部署的交互 demo。

## 💡 这个产品在解决什么

调研（4 场真实深访）得到的核心结论：学生选课的真痛点**不是"找不到信息"，而是"判断不了、比较不了"**——
信息散落在熟人私聊、共享表格、查分小程序里，口语化、好差评混杂、无法横向比较、时效性差、没人敢给定论。
而且**选课标准因人而异**：留学生看均分、保研看给分、求稳的看能不能通过——同一门课对不同人价值相反。

所以产品的价值主张是"**降低判断成本与不确定性**"，而非"帮你省几小时"。

**AI 真正非用不可的硬核点只有一处**：把口语化评价**结构化**成 13 个可比较字段（规则穷举不了，必须语义理解）。
其余（匹配度计算、统计展示、可解释排序）刻意用算法而非 LLM，保证可控、可解释、不黑箱。

---

## 🤖 Demo 里的四个 AI 触点

1. **实时结构化（F1）**：写一句口语评价 → AI 抽成 13 字段（给分/难度/作业量/收获/考勤/老师风格/适合人群…）逐项亮起。
2. **证据高亮**：每条 AI 结论旁标"引用自 N 条点评 / M 个客观字段"，可回溯到原始评价与客观数据。
3. **矛盾提醒**：当口碑说"给分好"但客观分布不支持时，AI 主动提示风险（防口碑虚高）。
4. **ChatBI 问数**：自然语言问"哪门课适合保研冲给分" → 识别条件 → 命中课程。

# 竞品 1 ｜ USTC 评课社区（icourse.club）

> 来源：网络检索 + 用户提供的 9 张截图（course/25072 数值分析、course/19509 数学分析B1、course 列表页、stats 站点统计、stats/rankings 排行榜）。截图日期至 2026-06-09。

## 1. 基本信息与规模
- **链接**：icourse.club（2.0 全球版 iCourses.com）
- **范围**：中科大单校起家；2025-02 上线 2.0 全球版，已导入 UCSD / UCB / UCI / UCSC / USC 五所美国大学，每校独立子站。
- **起始**：2015 年，源于一门《新生"科学与社会"研讨课》的课程项目；创始 张静宁 / 李博杰 / 常震。
- **规模（站点统计页，2026-06-09）**：运行 4043 天 / 用户 22,060 / 课程 18,635 / 点评 46,679 / 注册教师 1,578 / 平均评分 7.76 / 平均每门课 8.48 条点评。（对比 2025-02：用户约 1.42 万 → 16 个月增长约 55%。）
- **数据形态（关键，见第3节）**：点评数与用户贡献均为**陡峭长尾**——多数课程仅 1–2 条点评、多数用户仅写过 1 条；评分**严重正偏**（集中在 8–10 分）。即"规模化的成功样本"内部仍极度稀疏 + 好评偏差。
- **关键事实**：运营 11 年、原班人马持续维护，并主动向多校扩张——**这块"评课社区"赛道有活跃且在扩张的占位者**。

## 2. 数据采集与评价结构（用户怎么写一条评价）
- **每条评价含**：① 总体星级（5 星制，可半星，如 4.5）；② 4 个标准化维度下拉式打分——**课程难度 / 作业多少 / 给分好坏 / 收获大小**（取值如 中等 / 一般 / 超好 / 很多）；③ 一段自由文本（长度不限，常含给分细节、考试范围、考核形式等）；④ 学期标注（如 2026春）。
- **可附资料**：评价可附上传文件，如课件 PDF（截图 1："数值分析PPT 2026.pdf"）。
- **身份**：登录学生用户体系；支持匿名发布（匿名评不显示用户名、不进用户主页）。
- **❗缺口事实**：结构化维度**只有 4 个，无"点名/考勤"独立字段**；点名信息只能出现在自由文本里。"个人作业 vs 小组作业"、"评价者画像/目标"均无结构化字段。
- **观察到的用户行为（事实，非平台功能）**：有用户在自由文本里**自发手搓加权评分体系**（如"课程质量20% / 教学质量40% / 助教20% / 给分20% → 加权平均"）。平台不提供此功能，是用户自创——解读见战略判断。

## 3. 课程页呈现（聚合层）
- **课程级聚合评分**：总体星级 + 总分（10 分制，如 9.9）+ 评价人数（如 52 人评价）。
- **4 维聚合展示**：把所有评价的 4 个维度聚合显示（如 难度:中等 / 作业:中等 / 给分:超好 / 收获:很多）。
- **课程元数据（疑似取自教务）**：学分、课程号、开课单位、课程层次、选课类别、教学类型。
- **培养方案集成**：课程页"所属培养方案"为**级联导航**——学院 → 专业培养方案 → 年级（如 物理学院 → 应用物理学专业培养方案 → 2025级 数学通修1秋）。即同一门课被挂进多个专业/年级的培养方案，可反查"某专业某年级该上哪些课"。
- **AI 总结**：✅ 有，**且为分板块结构化总结**（按"教学质量""作业与习题"等板块组织，非单段文字），页面标注"AI 总结为根据点评内容自动生成，仅供参考"。
- **教师信息**：教师主页区块（姓名、头像、主页链接占位）。

## 4. 比较与筛选能力
- **同课多老师比较**：✅ 有。课程页右侧栏列出"其他老师的同名课"，每个老师附 总分 + 评价数 + 开课学期（如 数学分析B1：程艺 9.9(49)、李娟 10.0(32)、吴健 9.9(27)…）。
- **❗比较的粒度**：仅按**单一聚合总分**横向排列，**不支持按多维度筛选、不支持按用户自定义权重重排**。
- **课程列表筛选（粗粒度）**：仅按 课程类别（全部/通识/体育/英语/慕课/双学位/研究生课）筛选 + 按 课程评分 或 点评数量 排序。**无按给分/难度/作业量等维度的筛选**。
- **排行榜 + 排序算法（重要）**：站点有"排行榜"（最受欢迎课程/老师），默认排序用**归一化平均分**——`(本课程总分 + 全站均分×全站均点评数)/(本课程点评数 + 全站均点评数)`，即贝叶斯收缩，给每门课先垫约 8.5 条 7.8 分的虚拟点评，**已妥善处理"1 条 10 分 vs 10 条 9.9 分"的小样本偏差**。教师排行另设门槛（≥3 门课>9 分且无课<8 分）。→ **它的排序统计方法已相当成熟，不是裸均分。**
- **数据可视化**：站点统计页用 Highcharts 呈现 评分分布 / 课程点评数分布 / 用户贡献分布等**站点级描述性图表**（非面向个人选课的决策视图）。

## 5. 个性化
- **无**。AI 总结与聚合评分对所有访问者一致；无按人群/画像区分的视图，无"对你而言"的个人化建议。

## 6. 互动与社区机制（飞轮相关 → 喂运营方案参考）
- **每条评价下**：点赞（如 👍4）、回复/评论、复制链接分享。
- **课程级**：关注（计数）、推荐 / 不推荐（计数，如 推荐33 / 不推荐1）。
- **隐私/安全机制**：匿名发布；点评仅登录学生可见；用户可隐藏/取消隐藏自己的点评；投诉点评入口。
- **其他**：公告栏、站点统计、社区规范、发现漏洞（开源协作氛围）。
- **关键事实**：**贡献激励主要靠"点赞 + 被看见"的社交正反馈**——这是它对"为什么有人愿意写"给出的实际答案。

## 7. 时效性机制
- 评价按**学期 + 具体老师**组织（每条标学期；同名课按老师拆分）。课程页头部列出该课开过的多个学期。→ 评价天然绑定到"某学期某老师"，具备时效性结构。

## 8. 覆盖范围与冷启动方式
- 单校深耕 10 年起家；靠 科大强工科社区文化 + 创始团队长期维护活下来。
- 2.0 多校扩张采用"按学校导入课程/教师列表 + 独立子站"的方式起步。

## 9. 客观功能缺口（事实陈述，机会判断见下）
- 无"点名"等独立结构化字段（埋在文本）。
- 无多维筛选 / 无按个人权重重排。
- 无评价者画像、无人群标签、无个性化建议。
- 比较停留在单一总分维度。
- **⚠️ 不是缺口（别误记）**：排序算法（已用归一化/贝叶斯收缩处理小样本）、数据可视化（站点级图表已有）。这两项 USTC 已做，不能当差异化。

## 【战略判断 · USTC】（主观，非事实，agent 区别对待）

> 以下是我们对 USTC 的战略判断与定位结论，从上述事实推导而来，**但属于判断**。涉及"我们的产品该做什么"的设计决定**不在此处**，记在文档B。

1. **不做 USTC 翻版，做"适配评课荒地学校"的产品**。USTC 是参考标杆，不是要正面争夺的地盘——它的护城河是 3.7 万条点评的内容网络，正面拼内容必输（它有存量、我们从 0 起步）。
2. **"荒地学校多 = 机会"这个等式有风险**。评课社区技术门槛极低（2.0 导 5 所美国大学为证），别校 10 年没长出来，大概率不是没人会做，而是**冷启动飞轮转不起来**（无种子内容→无人看→无人写→死）。USTC 能活靠的是特定社区文化 + 创始团队 10 年维护。→ **荒地很可能是坟场。**
3. **定位的代价：命门从"功能差异化"转移到"冷启动"，且多校扩张让冷启动难度翻倍**（每进一个新校飞轮从零重启）。这是运营方案（交付物3）的头号问题，当前是整个方案中风险最高的一块。
4. **可借鉴（学机制，非抄功能）**：USTC 用"点赞 + 被看见 + 推荐/不推荐 + 匿名降门槛"驱动贡献——直接喂我们冷启动设计（种子内容 + 被问场景沉淀 + 社交正反馈 + 轻量模板）。
5. **我们相对它的真实增量（事实层面成立、措辞需精确）**：① 结构化维度是它的**超集**（多了点名/个人vs小组/评价者画像）；② 比较从"单一全局总分"升级为"**按对你重要的维度、可按你的权重重排 + ChatBI 问数**"——差别在**个性化**，不在算法质量或可视化。
6. **⚠️ 必须收紧的两处措辞（否则会被问穿）**：
   - **不能讲"我们排序更聪明 / 更会处理少评价的课"**——USTC 已用归一化平均分（贝叶斯收缩）解决了小样本偏差。
   - **不能讲"我们做数据可视化所以更强"**——USTC 已有站点级 Highcharts 图表。我们的可视化必须落在"**面向个人选课的决策视图（按权重多维对比 + ChatBI）**"，而非泛泛的图表。
7. **冷启动是命门 · 实证支撑（来自站点统计图）**：① 用户写点评数分布陡峭长尾——多数用户仅写过 1 条 → **"看强写弱"被实证，贡献极度集中**；② 课程点评数分布陡峭长尾——多数课仅 1–2 条 → **即便 11 年仍极稀疏**；③ 评分严重正偏（集中 8–10 分）→ **好评偏差（簇7 风险）被实证**。结论：连最成功的样本都如此，**飞轮/冷启动才是约束，不是功能**。
8. **需求信号（来自用户行为，非 USTC 功能）**：有用户在自由文本里手搓"加权评分体系"——**用户痛到自己发明权重打分，是我们"个性化按权重重排"最直接的需求证据**。

# 竞品 2 ｜ UST SPACE（ust.space · HKUST 科大社群）

> 来源：用户提供的 4 张截图（首页 /review、课程页 ACCT2010、单条评价 HUMA3800/view、左栏 BIEN 目录）。截图日期至 2026-06。

## 1. 基本信息与规模
- **链接**：ust.space／review；品牌"UST SPACE｜科大社群"。
- **范围**：HKUST 单校。
- **规模（部分可见）**：单门热门课点评量很高（ACCT 2010 = 1476 Reviews）；目录可见每门课点评数（BIEN 1010 = 77，多数课 1–10 条）→ 仍是头部密集、长尾稀疏。
- **关键事实**：成熟、功能丰富，部分结构化程度**超过 USTC**；是**设计者本校学生正在使用的工具**。

## 2. 数据采集与评价结构（用户怎么写一条评价）
- **每条评价含**：① 总体字母等级（A-/B+ 等，A–F 制）；② **4 个结构化维度**——CONTENT / TEACHING / GRADING / WORKLOAD，各给 A–F 字母等级 + 颜色编码（如 TEACHING=E 标红）；③ **考核构成结构化清单**（逐项 ✔/✗）：Mid-term Exam / Final Exam / Quiz / Assignment / Essay / Project / Attendance / Reading Material / Presentation；④ 自由文本，**按 CONTENT / TEACHING 等方面分栏**书写；⑤ 学期 + 讲师标注。
- **❗与 USTC 的关键差异**：维度集不同（无"难度/收获"，有"内容/教学"）；**考核构成做成了结构化字段**（USTC 埋在文本）；"点名"以 Attendance 进入考核清单。
- **缺口事实**：无"个人作业 vs 小组作业"区分；无评价者画像/目标字段。

## 3. 课程页呈现（聚合层）
- **聚合**：右侧面板显示总评论数 + 4 维聚合字母等级（如 CONTENT B+ / TEACHING B / GRADING B / WORKLOAD B）。
- **课程元数据**：学分（3 credits）、课程描述、**Offered In 开课学期史**、**Exclusions 互斥课**、Common Core 分区标注（如 "listed in Common Core (H)"）、**选课资格提示**（"You can enroll in this course"，账号感知）。
- **AI 总结**：❌ 未见（所示课程页无 AI 总结板块；置信度中）。

## 4. 比较与筛选能力
- **课程页内筛选/排序**：可按 讲师(All Instructors) / 学期(All Semesters) / 评分(All Ratings) 筛选，按 Popularity 排序，Reset All。→ 粒度是**单门课内筛评价**。
- **跨课比较**：首页有 Most Popular Courses / Top Rated Courses 榜单；**无跨课多维筛选、无按个人权重重排**。
- **目录视图**：左栏 School→Dept 展开后，每门课显示 点评数徽标 + ✓/✗ 标记（推测为选课资格，待确认）。

## 5. 个性化

## 【战略判断 · UST SPACE & 跨竞品】（主观，非事实）

1. **它直接挑战"USTC 是孤例、多数学校是荒地"的定位前提**：USTC、HKUST、（标签可见）THU、ZJU 四所顶校都有成熟评课社区，且 UST SPACE 结构化程度超过 USTC。→ **修正判断：能养活评课社区的学校基本都长出来了；"荒地"= 飞轮转不起来的学校，不是"碰巧没做"的学校。** 这不否定"适配荒地学校"的定位，但把命门进一步焊死在**冷启动**。
2. **设计者本校用户已有强工具**：科大学生（最易触达的真实用户）已在用 UST SPACE → 我们的产品对他们无增量价值。真实落地的目标用户只能是**没有这类工具、且我们能把飞轮转起来的学校**。
3. **⭐【运营方案启发 · 竞品调研产出】冷启动的系统化解法**（来自 UST SPACE 的 Timetable Planner）：评课社区先天死结 = 低频（一年用两次）+ 启动即空壳（没数据→没人看→没人写）。UST SPACE 的破法不是"激励大家写"，而是**绑定一个高频、第一天就有用、不依赖他人贡献的选课/排课工具（Timetable Planner）**，让评价搭在高频工具上沉淀——与访谈中 321CQU"高频工具带流量"同构。
   - **两层结构（喂交付物3）**：第一层=载体工具（高频、空壳也有用，负责"流量与留存"，让飞轮转得动）；第二层=种子内容机制（被问场景沉淀 + 种子铺设 + 强负面触发 + 轻量模板，负责"第一批内容"，让飞轮启得动）。**两层缺一不可**——工具解决"转得动"，种子解决"启得动"。
   - **比"冷启动给激励"更产品化**：把运营从"求人写评价"升级为"做一个本身就刚需的选课流程，评价是副产品"。
4. **个性化差异化更稳**：USTC 与 UST SPACE 两个最强竞品**都无个性化/画像/按权重多维重排** → 我们三条轴（个性化重排 / 多维可比较 / ChatBI）的差异化在两强面前同时成立。
5. **schema 是设计选择，做超集合理**：两家 4 维各不相同（USTC：难度/作业量/给分/收获；UST SPACE：内容/教学/给分/工作量）；UST SPACE 的**结构化考核构成清单**值得借鉴（比 USTC 强）。
6. **AI 总结不是标配**：UST SPACE 这么成熟也未上 AI 总结 → AI 总结可做差异点之一，但别高估其壁垒（USTC 已证明可做、UST SPACE 证明可不做）。

## 【战略判断 · THU & 三家收敛】（主观，非事实）

1. **THU = 顶校飞轮失败的活样本，是"冷启动是命门"最硬的实证**：清华（五万人级、学生能力顶尖）评课社区仅 3,387 条点评、96.5% 课程零评价。→ **飞轮转不起来与学校实力/学生能力无关，是机制本身极难启动。** 你想服务的荒地学校条件只会更差。
2. **修正"顶校都有成熟社区"为强度光谱**：USTC/UST SPACE（强）→ THU（弱：存在但未达临界量）→ 某信息荒漠校（零，访谈）。结论不是二元"有/无"，而是**内容网络强度**连续分布。→ 真正的市场不是"没有社区的学校"，而是"**社区弱/未起来、且我们有办法把飞轮转起来**"的学校。
3. **THU 强化我们的结构化差异化**：它是"口语化评价完全未结构化"的原始态（满屏自由文本）→ 我们做结构化超集对它是碾压级增量，也再次坐实 AI 结构化价值。
4. **缺正反馈机制 ↔ 飞轮失败（关联，非铁证因果，置信度中低）**：THU 恰好缺了 USTC 用来驱动贡献的那套机制——无点赞、无"有用性"投票、无收藏等贡献者正反馈，也无文件分享降低"被看见/被需要"的价值感。叠加纯评课站（无高频工具带流量），**这很可能是它只攒下 3387 条、覆盖率 3.5% 的重要原因之一**。→ 反向印证我们的运营设计：**贡献者正反馈 + 高频工具 是飞轮的两个必要件**，THU 两件都缺。
4. **三家收敛结论（喂文档B + 交付物2/3）**：
   - **个性化（画像/按权重多维重排）= 三家全无** → 我们最稳的差异化主轴。
   - **结构化 schema 各家不同、深浅不一** → 做超集 + 借鉴 UST SPACE 的结构化考核构成清单。
   - **AI 总结非标配**（USTC 有、UST SPACE/THU 无）→ 可做但非壁垒。
   - **冷启动是所有人的命门，且唯一被验证的解法是"高频选课工具带流量"**（UST SPACE）→ 交付物3 主轴。

# 选课助手 · 运营推广方案（交付物3）

> **项目一句话**：面向高校学生的 AI 选课决策助手——把散落、口语化的课程评价结构化成可比较、可个性化的数据，结合客观选课/给分数据，为不同目标的学生给出可解释的选课建议。
>
> **本文档对应题目交付要求 3**："制定切实可行的推广运营方案，确保产品能够在用户中得到广泛推广和应用。"
>
> **一句话定调**：这个产品的成败在冷启动。本方案回答的不是"怎么打广告"，而是"**评价飞轮怎么转起来**"——在一个学生团队可执行的资源约束下，把每个运营机制讲清楚，并为每条机制给出访谈或竞品证据。凡标 🔬 的数字与机制为试运营假设，验证前不作为结论。

---

## 一、核心命题：这个产品的成败在冷启动

### 1.1 为什么这么说：功能可以照抄，飞轮抄不来

交付物2 的竞品对比收敛于一句话：**所有人的难点都不是功能，而是冷启动。** 本方案从这句话接着写。三个事实支撑这个判断：

- **USTC 评课社区运营 11 年、代码在 GitHub 开源**——技术上并不神秘。但十余年间，很少有学校成功复刻出同等生命力的站点。难点不在"缺程序员"，在"缺可持续的内容飞轮"：没有种子内容就没人看，没人看就没人写，没人写社区就空壳化。
- **连最成功的样本内部仍极度稀疏**：USTC 多数课程只有 1–2 条点评、多数用户只写过 1 条、评分严重正偏（集中在 8–10 分）。规模化的成功样本，长尾依然陡峭。
- **顶校也救不了飞轮**：清华的选课社区课程覆盖率仅约 3.5%（3,387 条点评 / 34,773 门课，截至 2026-06 抓样）。飞轮转不起来与学校实力、学生能力无关，是机制本身极难启动。

访谈侧给出同样的信号：四校受访者**全员愿看评价，但主动写的意愿普遍弱**——"我凭什么花这一分钟写"、"加入已有讨论远多于从零发起"、"被问才答"、"很雷才主动避雷"。需求是真的，供给不会自动出现。

### 1.2 两段冷启动：载体买时间，飞轮长壁垒

产品的主体价值是决策层、而不止于评课社区（前台形态仍是评课社区底座），这没有让冷启动消失，而是把它重构成**两段**：

```
   第一段 · 载体带流量与留存                    第二段 · 评价飞轮
   （产品设计题，交付物2 已给到 P0/P1）          （运营题，本方案主体，仍是难点）

┌─────────────────────────────┐        ┌─────────────────────────────┐
│ 选课规划工具（课表/冲突/学分进度）│        │ 把被留住的人变成评价贡献者：     │
│ + 客观数据（给分分布/选满率）    │  留存   │  · 被问场景沉淀               │
│                             │ ─────▶ │  · 私域内容公域化（种子）       │
│ 零评价也有用 → 第一天就有价值    │        │  · 近零摩擦模板 + 贡献正反馈    │
└─────────────────────────────┘        │          │                  │
              │                        │          ▼                  │
              ▼                        │ 结构化评价积累 → 个性化更准     │
   客观数据 = 入场券，非护城河            │ → 更多人来看 → 更多人被问、写   │
   （321CQU 早已做出同类数据）           │ （差异化层在这里长出来）        │
   → 作用是为评价飞轮买时间              └─────────────────────────────┘
```

第一段的路径已被反复验证：321CQU、求是潮、Coursicle、UST SPACE 的 Timetable Planner——活下来的选课类产品**没有一个是纯评课站**，全部绑着一个高频、第一天就有用、不依赖他人贡献的工具。

但要把第一段的作用说诚实：**客观数据是入场券，不是护城河。** 给分分布、选满率这类数据 321CQU 早已做出，只靠它做出来的是"又一个 321CQU"，零差异化。载体的真实作用是：让产品零评价时就有用、撑住留存，**为第二段的评价飞轮买时间**。第一段是第二段的引流坡道，不是替代品。

### 1.3 本方案的姿态：不假装飞轮容易

我们不声称有让飞轮必然转起来的办法——顶校样本已经证明没有这种办法。本方案做的是**系统性提高飞轮启动概率的机制组合**：每条机制都能回溯到具体的访谈信号或竞品事实，没有凭空发明的动作；并且全部按**学生团队、近零预算、无学校官方强配合**的资源约束来写——后文的获客与运营动作是选课群、院系群、KOC、表单这个量级的，不是需要投放预算或商务团队的打法。这既是诚实的边界，也是可执行性本身。

## 二、从哪开始：弱社区校试点 → 荒漠校扩张，强社区校走工具层

运营方案开篇必答的问题：第一仗在哪打。

### 2.1 学校不是两类，是一条强度光谱

交付物2 以"双轨定位"收尾：有强评课社区的学校做工具层，没有的先解内容供给。落到运营上，双轨是光谱的两端——竞品深拆给出的更精确图景是**内容网络强度的连续分布**：

| 学校类型 | 代表 | 内容现状 | 对运营意味着 |
|---|---|---|---|
| **强社区校** | USTC、HKUST（UST SPACE） | 评价密度高、绑定强工具（Timetable Planner）、占位者活跃且在扩张 | 不硬碰、不作第一战场；走工具层/插件层合作（见 2.4） |
| **弱社区 / 半荒漠校** | THU（评课覆盖率仅 ~3.5%）；访谈中的几所半荒漠校（有查分工具、无评课社区，评价靠共享表格/红黑榜私下流传） | 学生已有看评价的习惯、私域里已存在真实贡献，但公开内容稀薄、飞轮未成 | **第一试点战场** |
| **信息荒漠校** | 访谈中的某信息荒漠校 | 几乎无公开评课内容，全靠私传的红黑榜、手写笔记、聊天记录 | 机制验证后的扩张目标＝增量最大 |

由此而来的判断：**真正的市场不是"没有社区的学校"，而是"社区弱或未起来、且我们有办法把飞轮转起来"的学校。** 本节的进入次序直接从这句话推出。

### 2.2 进入次序：先在"弱而不是零"处验证飞轮，再去荒漠摘最大的果实

**第一步 · 弱社区校试点。** 理由有三：

1. **飞轮第二段在这里最容易验证。** 冷启动最难的不是"让人看"（弱社区校学生已有看评价的习惯），而是"让看的人写"。这类学校的私域里已经存在真实贡献——共享表格、红黑榜、师弟师妹问答——缺的只是沉淀和结构化，不必先从零制造"看评价"这个行为本身。
2. **增量清晰。** 已有查分工具的学校恰好验证了"学生要客观数据"这件事，但结构化评价、个性化决策完全空白——我们的差异化层在这里没有对手。
3. **没有占位者可硬碰。** 试错成本最低。

**选校标准（三条同时满足）**：
- ① 评价内容**弱但非零**——有可公域化的私域内容做种子；
- ② 选课机制制造**高 stakes 人群**——竞价、抢课、必修也要选老师（目标用户密度）；
- ③ **团队有真实触达网络**——低资源 GTM 的全部渠道（选课群、院系群、KOC）的前提是"进得去"。一个学生团队没有空降陌生学校做运营的资源，这条标准不是可选项。

**候选（按标准点名）**：访谈网络覆盖的学校里，**候选校 X**（受访者 B"四年都是看这个表格过来的"——共享表格文化就是现成的私域公域化原料；选课币竞价机制天然制造高 stakes；有受访者可作首批 KOC）与**候选校 Y**（团队母校、网络最厚；同类查分工具验证了客观数据需求，但结构化评价空白）最符合三条标准。🔬 实际选定待试点启动前确认各校网络的真实强度。

**第二步 · 荒漠校扩张。** 信息荒漠型学校两段都要从零做：先建客观数据载体解决"第一天有什么可看"，再起评价飞轮。这是难度最高、但**增量最直观**的场景——验证过的 playbook（机制、模板、激励、节奏，见 §六）带过去，而不是拿荒漠当试验田。

**强社区校 · 另一条线，不在试点次序里。** 对 USTC / HKUST 型学校，"散、旧"已被社区解决，我们的增量只剩个性化决策层——形态是社区之上的**工具层 / 插件层**，走授权与合作，现阶段只作对标和未来方向，不投入运营资源。

### 2.3 接入边界：无授权，不搬运

这条既是策略也是合规底线，单独写明：

- 对已有评课社区的学校，优先以工具层/插件层姿态接入；**仅在获得授权或达成合作的前提下**，对其评价内容做 AI 结构化与个性化解释的增值；
- **无授权则不搬运、不批量抓取**任何社区内容；仅允许用户自行粘贴/导入个人可见的内容，或使用明确开放许可的数据做演示；
- 弱社区试点校的已有内容同理：私域内容（共享表格、红黑榜）**经原作者授权后**公域化作种子；第三方平台上的内容按上一条边界处理；
- 客观数据沿用交付物2 已定边界：用历史/公开口径，不实时抓取教务系统。

### 2.4 与 demo 的关系：一个讲价值上限，一个讲去风险路径

主动消歧，避免"demo 演示信息荒漠场景，第一试点却选半荒漠型学校"被误读为矛盾：

- **demo 演示荒漠校的满功率价值**——在信息荒漠里，从"全靠私传红黑榜"到"结构化+个性化+可解释"，产品增量最直观。它回答"这个产品的价值上限有多大"。
- **GTM 第一试点选弱社区校**——在"内容弱但不是零"的地方用最低风险验证飞轮机制。它回答"我们怎么以最大概率走到那个上限"。

一个是终点的果实，一个是起点的路径。荒漠校不是被放弃了，而是被排在了"机制验证之后"——它正是验证完成后要去摘的最大果实。

## 三、第一段冷启动：高频工具换来第一批用户与留存

第一段要解决的问题很朴素：**评价还是零的时候，凭什么有人来、有人留。** 答案不能是"评价"本身——必须是零评价也成立的东西。

### 3.1 两个获客钩子：规划工具管"高频"，客观数据管"传播"

**钩子一 · 选课规划工具（课表 / 时间冲突检查 / 学分进度）。** 这是被反复验证的载体路径：321CQU、求是潮、Coursicle、UST SPACE 的 Timetable Planner，活下来的选课类产品全部绑着一个高频刚需工具。它第一天就有用、不依赖任何人贡献，负责把人**带进来并留下**。

**钩子二 · 客观数据（给分分布 / 选满率）。** 它的获取成本低（历史与公开口径数据一次性整理即可），而对学生的传达力极强——"这门课去年均分多少、几秒选满"，一张卡片胜过十条转述，天然适合在群里传播。需要说明一处口径：在产品开发优先级上，客观数据查询列 P1（demo 非首发功能）；但**开发节奏与运营启用节奏是两条时间线**——试点校运营从第一个选课周期起就把它顶上获客一线，因为它是零评价期里最有"可看性"的内容。

为什么钩子不能直接是评价？零评价期没有评价可看；而即便在弱社区试点校用种子内容起步（§四），覆盖面也撑不起获客主力。**钩子必须是"对所有人、第一天就成立"的价值。**

### 3.2 推广时机：跟着选课周期走，打"用的时刻"

访谈给出一条关键约束：**痛的时刻与用的时刻是错开的**——踩雷的痛发生在学期中（想退课却被名额/学分卡住），而产品被需要的时刻在下一个选课周期，中间隔几个月，痛感早被磨平。所以推广不能指望"痛的人来找工具"，必须把动作压进**用的时刻**：

**选课周期 = 课程信息公布后 → 退补选截止。** 注意它比"选课周"宽：真实的信息调研和决策预案多发生在正式选课开始前 1–2 周（受访者 B 连续三天、每天约两小时的调研就发生在这个窗口），这段预热期与抢课/退补选窗口同为投放高峰。一年只有两个这样的窗口，其余时间的推广性价比急剧下降——**资源全部押在选课周期内**。

### 3.3 渠道：只用学生团队进得去的地方

按"近零预算、无官方配合"的约束，渠道清单如下，全部是内容驱动而非投放驱动：

| 渠道 | 动作 | 为什么有效 |
|---|---|---|
| 选课群 / 院系群 / 班级群 | 选课周期内投"课程客观数据卡片"（给分分布+选满率+已有结构化评价摘要），附"一键查这门课"入口 | 内容即广告；群是学生选课信息的第一现场（访谈簇1：渠道以群和熟人为主） |
| 学长学姐 KOC | 从访谈网络与院系熟人启动 5–10 人 🔬；请他们把"被问到的回答"沉淀进产品再转发回群（与 §4.2 联动） | 熟人推荐是现有信息网络里**信任度最高**的节点——借用而非对抗这个结构 |
| 校园墙 / 树洞 / 小红书 | 发"某院系选课避坑与推荐（结构化整理版）"攻略帖，文末挂产品 | 公开渠道"广但杂、慢、易淹没"（访谈原话）——我们的帖子恰好以"结构化、不杂"为卖点 |
| 自然流量节点 | 迎新季（新生第一次选课最茫然）、出分季（情绪与话题高峰） | 蹭确定性的校园节律，零成本 |

**明确不做**：付费投放、地推物料、依赖学校官方渠道转发。不是不想，是不装作有这些资源。

### 3.4 留存：把人留到下一个选课周期

选课天然低频（一年约两次），诚实面对而非粉饰：

- **载体即留存器**：课表是学期中每周都要看的东西，学分进度在期末有刚性查询需求——这两个高频小工具制造非选课季的回访理由，让产品不至于"用完即弃"；
- **移动端首屏承接课表高频**：课表不是藏在深层工作台里的附属功能，而应在移动端放到最显眼的位置（今日课表 / 下一节课 / 本周冲突）。运营上它承担"每天回来一次"的入口价值——用户为课表回来，平台才有机会在出分、选课预热等节点顺手触达写评价与查课；
- **留存主指标定义在"周期到周期"**：上个选课周期的用户在下个周期回来的比例（🔬 首轮目标见 §六），而不是日活——对低频工具谈日活是自欺；
- **留存的真正抓手是心智**：首个周期内让用户至少一次体验"这里能查到别处没有的东西"（客观数据卡片或一条结构化评价）——这是下个周期回来的理由。

## 四、第二段冷启动：让留下的人开始写评价

这是全方案最核心的一节。交付物2 把三个问题留给了本文：**第一批内容怎么出现？第一批用户为什么愿意写？好评者凭什么也写？** 逐个回答。所有机制遵守同一条纪律：**第一阶段只做轻量机制，不承诺重型社区功能**（课程页 Q&A、论坛、私信等留待成熟后）。

### 4.1 第一批内容怎么出现：私域公域化，不是从零创作

冷启动最常见的误区是"求大家来写"。访谈给出了更好的起点：**真实贡献本来就存在，只是沉淀在私域**——B 的学校有一张流传四年的共享评价表格（"四年都是看这个表格过来的"）；受访者 D 直接抄朋友整理好的红黑榜；每个院系都有"师弟师妹来问、学长学姐回答"的聊天记录。所以种子内容的来源不是创作，是**搬运 + 结构化**：

1. **经原作者授权，导入已存在的共享表格 / 红黑榜**。AI 把表格条目和口语描述转成统一 schema——这恰好是产品核心能力（口语→结构化）的第一次实战，种子内容的生产过程本身就是产品演示；
2. **团队 + 首批 KOC 自写种子**：每人写自己真实上过的课，近零摩擦模板让一条评价的成本压到一分钟以内；
3. **为试点院系的高 stakes 课先铺"讨论容器"**：每门课一个轻量页面，客观数据打底（不是空页），让后来者是"往已有内容上加一条"，而不是"从零发起"——访谈 A 的原话："加入已有讨论远多于从零发起"。

🔬 种子期目标（试运营假设）：试点院系的必修与高学分课程**覆盖率 ≥60%、每课 ≥3 条结构化评价**，再开始群投放——保证用户第一次点进来不是空壳。

### 4.2 第一批用户为什么愿意写：把"被问"变成资产

访谈里写评价的真实触发器高度一致：**被问才答**。B 和 受访者 D 都是"师弟师妹来问，我就答"；0 号自访谈的原话是"对方问什么，我就答什么"——而"系统让我写评价"则是"我凭什么花这一分钟"。结论：不要造一个"写评价"的新行为，**把已经在发生的"答"沉淀下来**。

机制设计（定位为**运营机制**，产品侧只留轻量入口）：

- **被问场景收集**：选课周期内，KOC 和团队在选课群/院系群里收集真实出现的问题（"民诉选哪个老师""X 老师点名吗"）；答过的人被顺手邀请："把刚才的回答存进来，下届就不用再问一遍"；
- **一句话入口**：产品侧只做一个输入框——"回答关于这门课最常被问的问题"，用户说一句人话，**AI 自动结构化成多维评价**（v2 抽取已在 18 条对抗性测试样本上人工抽查达 99% 准确率）。写评价的负担被压缩到："说一句你本来就会在群里说的话"；
- **贡献正反馈**：用"被看见"回答"我凭什么花这一分钟"——"你的评价帮 N 人做了决策"、被采纳为某次推荐的理由、收藏数。这是 USTC（点赞+被看见）与 UST SPACE（"X of Y 人觉得有用"投票）已验证的机制；而 THU 恰好**两件都缺**（无点赞、无有用性投票、无收藏），是飞轮未起样本的反面印证；
- **激励的边界**：适度、小额、不直接发现金——贡献者标识、周期性"最有用评价"，以及与校内外店铺合作的消费券 / 兑换券抽奖均可使用；"贡献解锁查看"这类强制互惠机制**谨慎使用**，防止换来一堆低质灌水（受访者 C 提过要激励，但激励换来的供给质量必须监控，见 §六）。

### 4.3 好评者凭什么也写：平衡负面偏差

访谈与竞品同时指向一个真风险：**强负面才主动**。受访者 C 的原话是"很雷才主动避雷"；而被反问"怎么让好评的人也写"时，他答不上来。竞品侧 USTC 评分严重正偏说明评价生态会系统性失真。任其自然，平台早期会长成"避雷墙"——看起来有用，实际损伤可信度（全是差评的平台，让人不敢信任何课）。

对冲机制（不声称根除，只声称对冲）：

- **模板结构反情绪**：写评价同时问"适合谁 / 不适合谁"，而不是给一个情绪宣泄框——把"吐槽"导向"信息"；
- **出分后定向触达**：刚上完课、刚知道给分，是记忆最新鲜、情绪最中性的写作时机——好课的"给分超预期"的惊喜也恰好在此刻，是邀请好评的黄金窗口（节奏见 §五）；
- **出分后奖励机制**：出分后 72 小时内完成一条"有效评价"即可参与抽奖，奖品优先用校内外店铺的小额消费券 / 兑换券。奖励对象是"有效信息贡献"，不是好评；AI 先过滤纯情绪、提问、灌水，中奖评价再人工抽查，避免激励稀释供给质量；
- **"推荐"也有正反馈**：展示"这门课经你推荐被 N 人选了"——让写好评和写差评获得同等的被看见；
- **热门课定向补视角**：对评价已偏向单一人群的课，定向邀请不同画像的人补充（"已有 3 条保研视角，缺就业视角"）。

这是真难题，机制只能提高平衡概率；正负偏差比进入 §六 的健康度监控，失衡时人工干预。

### 4.4 热度与网络效应：单校打透，不撒胡椒面

受访者 D 给出最后一块拼图：她写不写，取决于"**看热度、受众广不广**"——写作意愿与"会被多少人看到"直接挂钩。这决定了资源分配方式：**集中全部运营资源在单一试点校、甚至先在单一学院打透**，制造局部的热度临界点（"我们院选课都在用这个"），胜过多校同时铺开的胡椒面打法。配套 social proof 的轻量呈现："本周期已有 X 人用它完成选课"、课程页"X 人正在关注这门课"。

> 四个机制合在一起回答了交付物2 留下的三问：内容从私域搬运而来（4.1），写的动机靠"被问"与"被看见"（4.2），好评靠模板结构与出分时机平衡（4.3），整体供给意愿被局部热度放大（4.4）。

## 五、运营节奏：跟着选课周期走

前两节的机制铺到时间轴上。**选课周期 = 课程信息公布后 → 退补选截止**；一年两个周期，每个周期是飞轮的一整圈。以下是一学期作战表（全部为学生团队低成本动作）：

| 阶段 | 学生在做什么 | 运营动作 | 对应机制 |
|---|---|---|---|
| **预热期**（信息公布 → 正式选课，约 1–2 周） | 真实调研高峰：问学长、翻帖子、做预案 | 客观数据卡片投进选课群/院系群；KOC 启动答疑并沉淀；发院系攻略帖；种子内容补铺到位 | §3.1 / §3.3 / §4.1 |
| **选课与退补选窗口** | 决策、抢课、换课，最高频使用时刻 | "一键查这门课"承接群内提问；时间冲突检查工具自传播；密集收集"被问场景"问题清单 | §3.1 / §4.2 |
| **学期中**（低谷期，不强求） | 上课；少数人踩雷 | 移动端今日课表/下一节课制造自然回访；讨论容器慢养内容；踩雷的强负面评价自然流入（少量、不刻意激发） | §3.4 / §4.1 |
| **期末 → 出分后** | 刚上完课、刚知道给分 | **黄金收割窗口**：定向邀请写评价（好/差平衡触达）；用小额消费券/兑换券抽奖提高响应率；"评价说给分好，实际分布如何"的交叉验证内容二次传播 | §4.3 |

两点说明：其一，**唯一不可错过的是预热期+选课窗口**——这是一年仅有的两次获客高峰，团队人力（课程作业语境下即 1–2 人）在此期间全部压上；学期中允许低投入运转。其二，**出分后触达是供给侧最重要的一招**：它同时解决"记忆新鲜"（时效性）、"情绪中性"（平衡偏差）、"有据可写"（知道给分了）三件事，成本只是一轮定向消息。

## 六、增长与复制：从试点到扩张，以及怎么知道飞轮转起来了

### 6.1 试点打透的判定：飞轮转没转，看自然供给

试点不以"用户数"论成败，以**供给侧是否自我维持**论成败。以下判定线全部是 🔬 试运营假设，不是已验证结论；它们的作用是给第二个选课周期末的 Go / No-Go 一个明确口径：

| 判定线 | 为什么这么定 | 说明什么 |
|---|---|---|
| **自然新增评价（非团队/KOC 种子）占当周期新增 ≥50%** | 飞轮的定义就是"不靠我们推也在转"；若新增主要仍来自团队和 KOC，说明社区还停留在人肉驱动 | 供给侧是否开始自我维持 |
| **试点院系高 stakes 课覆盖率 ≥70%** | 不追求全校覆盖，先保证目标用户点进关键课时多数不是空页；70% 是"多数可用、不过度承诺"的试点线 | 种子内容广度是否够支撑第一次使用 |
| **每课有效结构化评价中位数 ≥3 条** | 1 条只是个体故事，2 条容易互相打架；到 3 条才开始有基本交叉验证价值。用中位数而不是平均数，防止热门课把数据撑虚 | 供给深度是否够支撑比较和个性化 |
| **北极星用户数环比上升，且周期间留存 ≥40%** | 选课产品天然低频，应看周期到周期是否回来；40% 表示首周期用过的人中有相当比例在下次选课仍记得回来 | 载体和内容是否形成心智 |

达线 → 启动扩张；不达线 → 机制复盘（是模板摩擦、正反馈、还是触达时机失效），而不是盲目换学校。

### 6.2 扩张次序与 playbook 迁移：什么能带走，什么必须重做

| | 可直接复制 | 必须重做 |
|---|---|---|
| **荒漠校（信息荒漠型）** | 机制层全套：私域公域化、被问沉淀、出分收割、节奏表、模板与激励 | **先补第一段**：客观数据载体从零建（公开课表/培养方案/历史给分整理）——荒漠校连查分工具都没有，客观数据在这里反而是最大增量；KOC 网络重建 |
| **强社区校（USTC/HKUST 型）** | 不适用运营 playbook | 走**授权/合作谈判**：以工具层/插件层身份对接已有社区内容（接入边界见 §2.3）——这是商务动作，不是运营动作 |

强社区校的"工具层 / 插件层"必须说具体，否则容易被误解成空话。它不是再做一个评课社区，而是在既有社区之上补三类能力：

1. **个人决策侧边栏**：用户在已有课程页或课程列表上打开我们的工具，选择"保研 / 留学 / 求通过 / 怕点名"等目标后，系统把已有评价翻译成"对我是否合适"，并引用原始评价与结构化字段作证据；
2. **跨课程选课工作台**：用户把多门候选课加入清单，我们负责做时间冲突、学分约束、给分倾向、workload、个人目标之间的综合比较。强社区通常回答"这门课大家怎么看"，我们补的是"这几门课我该怎么组合"；
3. **社区维护者工具**：为已有社区提供 AI 结构化、旧评标灰、矛盾提醒、低质评价过滤、ChatBI 问数等后台能力，帮助站点把多年积累的自由文本变成可计算资产。

这里的"不可替代"不应吹成"强社区团队自己做不出来"——USTC / HKUST 型团队技术力很强。更准确的价值是：**低侵入、低迁移成本、保留原社区主权的即插即用决策层**。对方不用搬走内容、不丢品牌和用户关系，也不用从零维护 AI schema、prompt 评测、证据追溯和个性化权重；我们作为增值层接在他们已有内容网络上。

对扩张速度保持诚实：**每进一所新学校，飞轮从零重启一次**，且选校标准③（团队可触达）持续生效——扩张优先沿团队真实网络推进（访谈网络覆盖的四校是天然的前两批名单），不画"一年覆盖百校"的大饼。

### 6.3 指标体系：北极星 + 供给侧健康度

**北极星：每个选课周期内，完成 ≥1 次"有证据支撑的课程取舍行为"的目标用户数 / 占比。**

定义拆解：**目标用户** = 高 stakes 选课者（交付物1 核心洞察：痛感∝stakes）；**有证据支撑的取舍** = 在产品内查看课程证据（结构化评价、客观数据、交叉验证视图任一）后，完成选定/放弃/更换某门课的决策动作；**周期口径** = 课程信息公布后至退补选截止。

为什么是它：它直接度量价值主张（"对我而言可解释的决策"），而且**两段冷启动通用**——零评价期基于客观数据完成的取舍同样计入，不会出现"指标在第一段天然为零"的失真。

**供给侧健康度仪表盘**（飞轮的体检表，全部 🔬 待试运营标定基线）：

| 指标 | 看什么 | 警戒信号 |
|---|---|---|
| 高 stakes 课覆盖率 | 种子+自然供给的广度 | 长期停滞 → 沉淀机制失效 |
| 每课有效结构化评价数 | 供给深度（"有效"由 AI 过滤字段判定） | 中位数 <2 → 个性化匹配缺燃料 |
| 贡献者分布 | 长尾健康度 | 前 10% 贡献者产出占比过高 → 飞轮实为人肉驱动 |
| 正负评价偏差比 | 避雷墙风险（§4.3） | 显著偏负 → 启动定向补视角 |
| 周期间留存 | "下个选课周期还回来吗" | 低于自然流失线 → 载体价值不足 |
| 推荐采纳率 | 个性化解释被信任的程度 | 看了推荐但不采纳 → 解释层可信度问题 |

这张健康度盘本身就是一个数据产品：用 BI 仪表盘监控内容生态，与本作业的数据语境天然一致——点到为止。

## 七、风险与诚实边界

把路演会被追问的每一处，提前写成白纸黑字。

**1 · 飞轮可能起不来——这是真风险，不回避。** 顶校样本（覆盖率 ~3.5%）证明失败是常态而非意外。我们的对冲是结构性的：载体先撑住留存（人在，飞轮就还有机会）、被问沉淀把写作门槛降到一句话、出分后触达把供给时机选在最优点。同时设退守线：🔬 若两个选课周期后自然供给仍接近零，产品退守"客观数据工具 + 个人导入"形态——价值缩水，但不归零。

**2 · 负面偏差风险。** 机制对冲见 §4.3，监控指标见 §6.3；承认机制只能平衡概率，不能保证生态。

**3 · 内容治理与信任机制。** 评价类产品必被问"怎么防乱骂老师/刷评价"：匿名展示 + 校内身份校验（学校邮箱验证，一人一账号）；AI 结构化自带"有效评价"过滤（提问/闲聊/纯情绪宣泄不入聚合）；针对教师的人身攻击内容不入库，举报-下架通道常设；**AI 结构化保留原文证据**——每个结构化字段可回溯到评价原句，被质疑时有据可查，这同时是产品证据层的既有设计。结构化展示本身也是治理手段：把表达导向"给分/作业量/适合谁"的事实维度，而不是给情绪一个放大器。

**4 · 合规与接入边界。** 不实时抓取教务系统（交付物2 砍除清单既定边界）；客观数据用历史与公开口径；无授权不搬运、不批量抓取任何已有社区内容，种子内容仅来自原作者授权的私域材料、用户自行导入的个人可见内容、明确开放许可的数据（§2.3）。

**5 · 低资源执行假设。** 全方案动作以"学生团队、近零预算、无学校官方强配合"为前提——选课群、院系群、KOC、表单、内容运营，没有一项需要钱或公章。这既是局限的如实声明，也是可执行性的证据：方案里没有一步是"假设我们有商务团队"。若试点中获得院系层面的支持（如辅导员转发），视为加速器而非前提。

**6 · 口径护栏。** 本方案全程不说"靠客观数据解决了冷启动"——客观数据是入场券不是护城河，它的作用是让产品零评价也有用、为评价飞轮买时间。差异化壁垒只能由结构化评价资产长出来，而那必须靠本方案的第二段机制去挣。

**7 · 验证方式。** 本方案的全部目标数字标 🔬，验证动作与交付物1 的遗留问题（"用户会不会换掉现有免费渠道并持续用"）合并为同一件事：**demo 小范围真实试用，看采纳与留存**——试点校第一个选课周期就是验证场。

---

**收束**：这个产品的功能层已经收口（交付物2），demo 已经在路上（交付物4），真正决定它能不能活的，是本文回答的问题——飞轮怎么转起来。我们的答案不是一句口号，而是一组全部有访谈与竞品证据、且学生团队执行得动的机制：**载体买时间，私域变种子，被问变资产，出分收供给，单校打热度，指标看自然供给。** 冷启动没有必胜法，但有纪律——这份方案就是那套纪律。
